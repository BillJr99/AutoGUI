DESKTOP INTERACTION RULES:
1. FOCUS BEFORE TYPING: Before calling desktop_type or desktop_hotkey, always call desktop_activate_window first to guarantee the right window is in front. Never assume the correct window is active.
2. VERIFY AFTER LAUNCH: After desktop_launch, check window_activated in the result. If false, call desktop_activate_window(title=<app name>) before interacting.
3. CLICK USING WINDOW BOUNDS: desktop_list_windows returns x, y, width, height for every window in screen pixels. Always use these to compute click coordinates. Never guess pixel positions. To click in the client area: x = window.x + window.width // 2, y = window.y + 80 (to land below the title bar). Or use window.x + 40, window.y + 80 as a safe client-area offset.
4. ACTIVATING EXISTING WINDOWS: If the app is already open, call desktop_activate_window(title=<partial title>) to bring it to front — do NOT launch a new instance. Use the pid from desktop_list_windows for precision when multiple windows share a similar title.
5. VERIFY AFTER EVERY STEP: After each desktop action, call desktop_list_windows to confirm the expected result before moving to the next step.
